2. 任务类型与动作空间

数据集 任务层级 动作空间 维度 是否包含干预/自然演化分界 因果解耦适用性
PandaPush 桌面 Manipulation（推方块到目标） 连续: Δ(x,y,z) 末端位移 3 是 — 机械臂推动 vs 物块自然滑动/碰撞反弹 高
Sketchy 桌面 Manipulation（抓取/推/放置） 连续: 7-DoF 关节速度 7 是 — Sawyer 执行动作 vs 物体惯性运动 高
CLEVRER 物理推理（碰撞预测/反事实） 无动作（纯观测） 0 仅自然演化 — 无机器人干预 中（无主动因，但有碰撞因果链）
Balls Occlusion 物理模拟（弹性碰撞） 无动作 0 仅自然演化 低（简单动力学，无外部干预）
边界警示：CLEVRER 和 Balls Occlusion 不含机器人动作，无法验证"动作→环境因果解耦"假设。它们仅适用于验证"物体间因果传递"（碰撞/遮挡）。若要验证"agent-caused vs environment-caused"分界，必须使用 PandaPush 或 Sketchy。

3. 动力学与物理一致性

数据集 确定性/随机性 接触动力学 物理复杂度 深度交互丰富度
PandaPush 确定性（MuJoCo仿真） 刚体推、滑动、物块间碰撞
中等：平面推+碰撞
主要为平面运动，深度变化有限
Sketchy 确定性（仿真，PyBullet） 抓取、放置、推、接触力丰富
高：多物体+抓取+堆叠
有深度变化（抬起、堆叠）
CLEVRER 确定性（Blender物理引擎） 弹性碰撞、滚动
中等：碰撞+摩擦
3D场景，有前后深度关系
Balls Occlusion 确定性（2D弹性碰撞模拟） 弹性碰撞（简单）
低：仅2D碰撞
无深度（纯2D）
物理一致性总结：所有数据集均来自确定性物理引擎，同一初始状态+动作产生唯一下一状态。这简化了世界模型的训练目标（无需建模aleatoric uncertainty），但也意味着在真实世界部署时需额外考虑随机性建模。
失效场景警示：
  • Balls Occlusion：仅2D运动+弹性碰撞，不适合验证"3D理解"或"深度依赖的因果推理"
  • PandaPush (1C/2C)：物体在平面运动为主，深度变化有限。验证3D优势需聚焦遮挡场景和多物体堆叠
  • 所有数据集均缺乏形变和流体场景 — 不适合验证非刚体动力学

4. 因果结构标注

数据集 主动因素 (Agent) 被动因素 (Env) 反事实数据 因果链可追踪性
PandaPush 显式 — action Δ(x,y,z) 每帧记录，与 EE 位移严格对应 物块间碰撞反弹（隐式，可从轨迹推断） 无 高 — 可通过仿真重放生成反事实
Sketchy 显式 — 7D关节速度+抓取标记 重力导致物体下落（隐式） 无 高 — 丰富的状态标注便于因果推断
CLEVRER 初始速度（发射动作，非在线控制） 详细标注 — 碰撞事件、因果图 有 — 提供反事实问答（"如果移除物体X..."） 高 — 官方提供因果关系标注
Balls Occlusion N/A 弹性碰撞（简单物理规则） 无 中 — 可从GT位置推断碰撞因果
因果解耦验证路径：
  1. Agent→Object（机器人推动物体）：PandaPush + Sketchy 可直接验证
  2. Object→Object（碰撞传递）：CLEVRER 最丰富（含官方因果图标注）
  3. 反事实推理：仅 CLEVRER 原生支持；PandaPush/Sketchy 可通过仿真重放构造
边界提示：PandaPush 和 Sketchy 仅有正向轨迹（成功的 expert 策略），无失败样本。验证"解耦鲁棒性"需额外生成：(1) 随机策略轨迹 (2) 仿真重放的反事实配对。CLEVRER 的反事实标注是现成可用的。

5. 对象中心属性 (Object-Centricity)

数据集 实例标注类型 对象持久性追踪 部件级标注 与粒子表征的兼容性
PandaPush 隐式（方块颜色区分） 未显式标注 — 可通过颜色追踪 无（刚体方块无部件） 高 — 物体数固定，方便 particle-to-object 对应
Sketchy 隐式（物体形状/颜色） 未显式标注 — 需自行建立 无 高 — 物体少且刚性，适合稀疏粒子
CLEVRER GT masks + 属性（形状/颜色/材质） 唯一ID跟踪，跨遮挡保持 无 高 — 完整GT便于量化 ARI/tracking accuracy
Balls Occlusion GT 位置 + ID 唯一ID，设计用于测试遮挡追踪 无 高 — 直接可对比 particle vs GT ball
粒子表征演化验证：
  • CLEVRER + Balls Occlusion：有GT实例标注，可直接计算 ARI、Tracking IoU
  • PandaPush + Sketchy：无GT mask，依赖 obj_on (z_t) 和颜色聚类间接评估
  • 遮挡恢复：Balls Occlusion 专门设计用于此测试（P3 预实验的理想平台）
